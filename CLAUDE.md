## ⚠️ Важная особенность файла index.html — прочитать перед любыми правками

`index.html` экспортирован из Claude Design и **не является обычным статическим HTML**. Внутри есть служебные конструкции вида `__bundler/manifest`, `__bundler/template`, кастомные теги (`<x-dc>`, `<helmet>`). Это значит, что видимая на экране разметка может собираться JS-скриптом во время загрузки страницы, а не лежать как обычные `<section>`/`<div>` напрямую в файле.

**Перед тем как вставлять форму регистрации:**
1. Сначала разберись, где на самом деле находится редактируемый контент (возможно, внутри `<script type="__bundler/template">` как строка, а не как живой DOM)
2. Не пытайся искать место для вставки по обычным CSS-селекторам, пока не подтверждено, что разметка не генерируется на лету
3. Файл подтверждён рабочим (открывается и в Edge, и в Chrome, с интернетом и без) — это просто непривычная для обычного редактирования структура, не баг

## Архитектура

- **Frontend:** `index.html` — экспорт из Claude Design, самодостаточный (шрифты и картинки встроены), но с нетривиальной внутренней структурой (см. выше)
- **Backend:** `apps-script/Code.gs` — Google Apps Script, разворачивается как веб-приложение (`doPost`). **Уже настроен и работает** в Google-аккаунте пользователя — сам файл здесь лежит как референс для Claude Code, а не как то, что нужно создавать заново
- **Хранение данных:** Google Таблица (заявки) + папка на Google Диске (чеки об оплате) — уже созданы
- **Связь:** форма должна отправлять `fetch POST` (Content-Type: text/plain, чтобы избежать CORS preflight) на URL развёрнутого Apps Script — этот код уже есть и проверен в старом проекте `neuro-smeno-2026-08/index.html`, нужно перенести ту же логику, а не писать с нуля

## Git-воркфлоу

Работаем через ветки и Pull Request, не пушим напрямую в `main`.

```bash
git pull origin main
git checkout -b feature/название-изменения
# ...правки...
git add -A
git commit -m "feat(scope): краткое описание"
git push origin feature/название-изменения
```

Дальше: PR → мердж → подождать 1–2 минуты (GitHub Pages пересоберётся) → проверить в окне инкогнито.

Формат коммитов: `тип(область): описание`, например `feat(form): добавить секцию регистрации в лендинг Claude Design`.

Для цен/дат/текста согласия — только через ветку/PR, не напрямую в `main`.
