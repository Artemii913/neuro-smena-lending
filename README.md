# Нейро-мультипликационная смена — лендинг (Claude Design) + форма регистрации

## Структура проекта

```
Neuro-smena-lending/
├── index.html          ← экспорт из Claude Design (переименован с кириллицы)
├── CLAUDE.md
├── README.md
└── apps-script/
    └── Code.gs          ← референс backend-кода (сам Apps Script уже живёт в облаке)
```

## Backend уже настроен — повторно создавать не нужно

Google Таблица, папка на Диске и Apps Script уже созданы в прошлом проекте. Здесь нужно только:
1. Открыть тот же Apps Script проект (через Google Таблицу → Расширения → Apps Script)
2. Если ещё не развёрнут — **Развернуть → Новое развёртывание → Веб-приложение** (Execute as: Я, Who has access: Все)
3. Скопировать ссылку `.../exec`

## Шаг 1 — Разобраться со структурой index.html

Прежде чем вставлять форму, нужно понять, как Claude Design хранит контент внутри файла (см. предупреждение в CLAUDE.md про `__bundler/template`). В Claude Code это можно начать так:

```
Прочитай index.html в этой папке. Объясни, где на самом деле находится
основной контент страницы — это обычный HTML в <body>, или он собирается
через JS из строки внутри одного из <script> тегов (ищи
__bundler/template, __bundler/manifest)? Не вноси изменений, просто
объясни структуру.
```

## Шаг 2 — Перенести форму регистрации

В старом проекте (`neuro-smeno-2026-08/index.html`) есть рабочая секция формы с id="register" и JS-логика отправки. Попроси Claude Code:

```
В папке neuro-smeno-2026-08 (или просто вставь содержимое файла в чат)
есть старая версия сайта с рабочей формой регистрации — поля имени
родителя и ребёнка, загрузка чека, чек-бокс согласия на обработку
персональных данных, и JS, который отправляет всё это через fetch
на Google Apps Script (с Content-Type: text/plain, чтобы избежать
CORS preflight).

Перенеси эту секцию и эту JS-логику в текущий index.html (лендинг
из Claude Design), сохранив стиль и цвета текущего лендинга, а не
старого сайта. Не меняй текст полей и логику отправки.
```

## Шаг 3 — Подключить Apps Script

Вставить полученную на старте ссылку `.../exec` в константу `APPS_SCRIPT_URL` в перенесённом JS-коде.

## Шаг 4 — Git и GitHub

```bash
git init
git add -A
git commit -m "feat: лендинг из Claude Design + форма регистрации"
git branch -M main
git remote add origin https://github.com/ВАШ_АККАУНТ/НАЗВАНИЕ_РЕПО.git
git push -u origin main
```

Дальше — Settings → Pages → Source: Deploy from a branch → main → / (root).

## Проверка перед запуском

- Открыть итоговую ссылку, отправить тестовую заявку с тестовым файлом
- Проверить, что строка появилась в Google Таблице, а файл — в папке на Диске
- Проверить на мобильном
- Проверить, что форма не сломала анимации/вёрстку остального лендинга
- Открыть в инкогнито — не в обычном окне (кэш)

## Частая ошибка

Если при локальном тестировании форма не отправляется — открой страницу через `npx serve` или расширение VS Code «Live Server», а не как `file://` напрямую — некоторые fetch-запросы блокируются браузером для локальных файлов.
